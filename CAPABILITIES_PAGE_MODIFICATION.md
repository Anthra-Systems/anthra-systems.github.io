# Capabilities Page Modification Brief

## Page

Capabilities Page

## Current Issue

The Capabilities page currently feels like a plain list of engineering skills.

It needs to feel more premium, visual, and client-understandable.

The page should not only say what Anthra can do. It should show how those capabilities turn into real client outputs and link visitors toward relevant solution pages.

## Goal

Redesign the Capabilities page so it communicates Anthra’s engineering depth in a more visual and practical way.

The page should show:

* what Anthra is technically capable of building
* how capabilities combine into complete systems
* what clients actually get from each capability
* which existing solutions are related to each capability

Tone:

* premium
* practical
* engineering-led
* client-friendly
* not too technical
* not a plain service list

## Important Restrictions

* Do not generate new images now.
* Use only existing relevant images/assets already available in the project.
* Do not create new solution pages.
* Do not create duplicate routes.
* The project already has 25 existing Solution Detail pages/routes.
* Every related solution link must use existing solution data/routes only.
* Do not invent new slugs.
* If a related solution is not found, skip it or map it to the closest existing solution.

# Additional Animation and Image Direction

## Auto Animation Requirement

Do not rely only on hover or click interactions.

Where suitable, add subtle automatic animation so the page feels alive even if the visitor does not interact.

Use auto-animation carefully and professionally.

Allowed auto-animations:

* slow auto-changing cards
* auto-advancing active step in a flow
* slow horizontal card movement
* soft image pan/zoom inside cards
* animated connector line
* subtle pulse moving through a process flow
* rotating highlight between capability/R&D cards
* automatic reveal of quote lines or process stages

Interaction can still exist, but auto-animation should make the section engaging by default.

Important behavior:

* pause auto-animation on hover or when user interacts
* allow click/tap to manually select a card or step
* do not make motion too fast
* respect reduced-motion preferences if possible
* avoid distracting, flashy, or childish animation

Preferred timing:

* auto-advance every 4–6 seconds
* slow image motion / pan should be subtle
* connector animation should be calm and premium

## Image Usage Direction

Images may be used where they improve clarity and visual quality.

Use images to make cards and sections more practical, understandable, and implementation-focused.

Allowed:

* use existing project images/assets
* crop existing images
* use image-backed cards
* darken or overlay images for readability
* use subtle image backgrounds inside cards
* use image strips or preview panels
* generate new images only if existing images are not suitable

If generating new images is required, follow Anthra visual constraints:

Image style:

* realistic
* practical
* implementation-focused
* clean
* premium but not overly polished
* understandable for novice clients

Avoid:

* fake holograms
* sci-fi visuals
* random corporate people
* posh corporate campus images
* overly polished AI-render look
* generic stock imagery
* excessive orange lighting
* too dark or dirty industrial scenes

If direct image generation is not available inside the workflow, create or update `IMAGE_GENERATION_PROMPTS.md` and list:

* image needed
* page/section where it will be used
* exact generation prompt
* preferred filename
* status: pending

Images should support the meaning of the section, not just decorate it.

Every image should answer:

“What real-world system, capability, prototype, or engineering process is this showing?”
