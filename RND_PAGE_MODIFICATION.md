# R&D Page Modification Brief

## Page

Research & Development / R&D Page

## Current Issue

The current R&D page has the right direction but still feels too plain, dark, and static.

It currently looks like a list of dark cards placed one below another. The page communicates R&D, but it does not yet feel like a premium applied engineering lab page.

Main issues:

* too many small dark boxes
* weak visual hierarchy
* limited interactivity
* too much empty black space
* R&D philosophy quote is useful but visually needs more premium treatment
* approach flow is too simple and not engaging enough
* prototype areas look like normal cards instead of practical R&D directions
* the page does not yet strongly show Anthra as an applied engineering and R&D company

## Goal

Redesign the R&D page so it feels like a premium applied engineering and R&D page for Anthra Systems.

The page should communicate:

* Anthra studies real-world engineering problems.
* Anthra experiments with technical approaches.
* Anthra builds prototypes before full deployment.
* Anthra tests systems in practical conditions.
* Anthra turns uncertain ideas into field-ready intelligent systems.
* Anthra is not just a development/manufacturing vendor, but an applied engineering lab.

Tone:

* premium
* serious
* engineering-led
* practical
* R&D-focused
* client-understandable
* not overly academic
* not generic startup language

## Visual Direction

Use Anthra’s premium dark engineering theme, but avoid making the page visually flat.

Use:

* dark graphite background
* dark glass / slate cards
* controlled Anthra orange accents
* subtle engineering grid/linework
* existing relevant lab/prototype/testing images
* image-backed cards where useful
* interactive flows
* clear spacing and hierarchy

Avoid:

* plain stacked dark boxes
* too much pure black
* excessive orange
* fake holograms
* sci-fi visuals
* generic corporate office photos
* random stock engineers
* overly polished AI-generated look
* too many small unreadable cards

## Image Handling

Do not generate new images immediately.

Use existing available images/assets where suitable.

Allowed:

* crop existing images
* darken images
* blur images slightly
* use image overlays
* use existing hero/R&D/prototype/lab images
* use subtle image-backed cards

If required images are missing, create structured placeholders in the code/data and mark them as pending.

Do not use random external stock images.

## Final Page Structure

The R&D page should follow this structure:

1. Hero
2. Anthra R&D Philosophy Quote
3. What We Explore
4. When R&D Is Needed
5. R&D Approach
6. Prototype Areas
7. From Question to Working System
8. Example R&D Directions
9. Final CTA

# 5. R&D Approach

## Purpose

Show how Anthra moves from uncertainty to a tested direction.

## Section Title

From Uncertainty to a Tested Direction

## Flow

Problem Study → Feasibility → Prototype → Testing → Field Trial → Productization

## Step Descriptions

1. Problem Study
   Understand the real-world challenge, constraints, users, site, machine, or operating environment.

2. Feasibility
   Explore what is technically possible, practical, and worth building.

3. Prototype
   Build a testable first version using suitable hardware, software, mechanics, or AI.

4. Testing
   Validate performance, reliability, limitations, and failure points.

5. Field Trial
   Test the prototype in practical operating conditions.

6. Productization
   Move toward a deployable, maintainable, and scalable system.

## Design

Current version is too plain.

Improve using:

* horizontal process flow
* active step hover
* animated connector line
* numbered stages
* compact explanation panel
* subtle orange flow pulse

Avoid making it look like a plain row of boxes.
